01

LegacyLink AI

Modernization & intelligence layer for Progress OpenEdge ERPs

Thousands of manufacturers and distributors still run their core operations on OpenEdge — 30-year-old enterprise software few engineers can read anymore. LegacyLink AI sits on top of an existing OpenEdge system, without replacing it, and adds a live dashboard, real-time anomaly alerts, and a natural-language query box that never lets an LLM touch raw production data.

  • Implemented the transactional outbox pattern directly in ABL triggers for change-data-capture, since OpenEdge has no native log-streaming connector.
  • Constrained the LLM to an allow-listed set of parameterized query templates via forced tool-use — a hijacked prompt can pick a template, nothing else.
  • Built real role-based access control: JWT sessions, PBKDF2 password hashing, self-service signup, admin-managed promotion, single-use password reset.
ABL / OpenEdge Python / FastAPI PASOE REST JWT auth Claude tool-use

02

Askforge

RAG-based document & web Q&A application

A genuine retrieval-augmented generation app, not a demo wrapper — built to answer questions grounded in real uploaded documents (PDF, Word, CSV, Markdown, text) and live web content, with support for multiple LLM providers so it isn't locked to one vendor's uptime.

  • Designed a multi-provider LLM architecture (OpenAI, Mistral, Google Gemini) with automatic detection and failover, avoiding dependency on a single AI vendor.
  • Built a Flask REST API with real-time document upload, chunking, and vector embedding into a persistent ChromaDB store — zero downtime knowledge-base updates.
  • Diagnosed and resolved production-breaking Python 3.14 compatibility issues in native extensions, then deployed to Railway with Gunicorn, persistent storage, and environment-based secrets.
